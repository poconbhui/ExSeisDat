4 Example: Hello, World! {#examples_hello_world_c}
5 =====================

9 In this example, we'll look at initializing a PIOL object, potentially
10 across multiple processes, and do some operation on just one of those
11 processes.

17 First, our boilerplate.
18 We'll include the ExSeisDat header, and any extras we need.
19 We also define a `main()` function.
20

22 // Code:
23 
24 #include "ExSeisDat/PIOL.h"
25 
26 #include <stdio.h>

31 @endcode
32 We initialize the ExSeis object, frequently referred to in this project
33 as the PIOL object.
34 @code{.c}
35 // Code:
36 
37  // Initialize the PIOL
38  PIOL_ExSeis* piol = PIOL_ExSeis_new(PIOL_VERBOSITY_NONE);
39 
40 @endcode
41 Here, we've initialized the PIOL with the lowest verbosity setting.
42 By default, this will initialize using MPI over MPI_COMM_WORLD.
43 That means any operations run using this PIOL object will run on all
44 the launched processes.
45 
46 Next, we'll run some code on just one process. In this case, we'll get
47 process 0 to print the number of processes available.
48 @code{.c}
49 // Code:
50 
51  // Run on rank 0.
52  if (!PIOL_ExSeis_getRank(piol)) {
53  // Get num ranks for the PIOL and print to stdout.
54  size_t num = PIOL_ExSeis_getNumRank(piol);
55  if (num == 1) {
56  printf("Hello, World! There is 1 process\n");
57  }
58  else {
59  printf("Hello, World! There are %zu processes\n", num);
60  }
61  }
62 
63 @endcode
64 Having process 0 do serial operations is a common occurrence in MPI,
65 and indeed many parallel codes. Process 0 is, then, often referred to as
66 the master process.
67 
68 Having done the work we set out to do, we will now delete the PIOL
69 object. This step is important to ensure files and caches are shut down
70 properly when the program closes.
71 @code{.c}
72 // Code:
73 
74  // Cleanup the PIOL.
75  PIOL_ExSeis_delete(piol);
76 
77 @endcode
78 And finally, return 0 to indicate success.
79 @code{.c}
80 // Code:
81 
82  return 0;
83 }
84 @endcode
